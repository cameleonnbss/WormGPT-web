## 🛠️ How It Works

1. **`install.bat`** detects your GPU VRAM, downloads a pre-compiled `llama.cpp` server, downloads the best Gemma 4 Uncensored Heretic model quantization for your memory, and installs Python dependencies.
2. **`start.bat`** launches the llama.cpp server in the background, waits for it to be ready, then starts the Flask web backend and opens your browser.
3. The **web UI** (`web/index.html`) talks to the Flask backend (`backend/server.py`), which proxies requests to the llama.cpp server on port 8080.

---

## 📥 What the Installer Does

| Step | Action |
|------|--------|
| 1 | Checks Python and Git are installed |
| 2 | Detects GPU VRAM (NVIDIA, WMI, dxdiag, or manual input) |
| 3 | Chooses best model quantization for your VRAM |
| 4 | Downloads pre-compiled llama.cpp server (no compiler needed) |
| 5 | Extracts server and all DLLs into `deps/` |
| 6 | Downloads the model from HuggingFace (`~5-6 GB`) |
| 7 | Installs Python dependencies (`flask`, `flask-cors`, `requests`, `huggingface_hub`) |
| 8 | Creates `start.bat` with correct context size |

### Quantization Selection

| VRAM | Quantization | Context |
|------|-------------|---------|
| < 8 GB | Q4_K_M | 2048 |
| 8–12 GB | Q5_K_M | 4096 |
| 12–16 GB | Q6_K | 4096 |
| > 16 GB | Q8_0 | 8192 |

## 🔧 Manual Server Launch (Troubleshooting)

If `start.bat` doesn't work, you can launch the two servers manually in **two separate PowerShell windows**:

### Terminal 1 — llama.cpp server
```powershell
cd "C:\path\to\WormGPT-web\deps"
.\llama-server.exe -m "..\models\gemma-4-E4B-it-ultra-uncensored-heretic-Q5_K_M.gguf" --host 127.0.0.1 --port 8080 -c 4096
```
*(Replace the model filename with yours — check `models\` for the exact name.)*

Wait until you see `HTTP server is listening` and `model loaded` before opening Terminal 2.

### Terminal 2 — Flask backend
```powershell
cd "C:\path\to\WormGPT-web\backend"
python server.py
```

Then open **http://localhost:5000** in your browser. Both terminals must stay open while you chat.

## 🚨 Common Errors & Fixes

### 1. `Python not found`
**Cause:** Python is not installed or not in PATH.
**Fix:** Install Python 3.10+ from [python.org](https://python.org). **Check "Add Python to PATH"** during installation. Restart your terminal after installing.

### 2. `Git not found`
**Cause:** Git is not installed.
**Fix:** Install from [git-scm.com](https://git-scm.com). Choose "Git from the command line" during installation.

### 3. `CMake not found` / Build fails
**Cause:** No C++ compiler available.
**Fix:** The installer now uses a **pre-compiled** llama.cpp server — no CMake or compiler needed. If you see this error, download the latest installer version.

### 4. Model download fails (404 Not Found)
**Cause:** Wrong HuggingFace repo or filename.
**Fix:** The installer uses `mradermacher/gemma-4-E4B-it-ultra-uncensored-heretic-GGUF`. If it fails, check that the repo still exists at: https://huggingface.co/mradermacher/gemma-4-E4B-it-ultra-uncensored-heretic-GGUF

### 5. `unknown model architecture: 'gemma4'`
**Cause:** llama.cpp version is too old to support Gemma 4.
**Fix:** The installer now downloads a recent release (`b9222` or newer) that supports `gemma4`. Delete `deps\llama-server.exe` and re-run `install.bat`.

### 6. `ggml-base.dll not found` / DLL popups on launch
**Cause:** The server binary was copied without its DLL files.
**Fix:** The installer now uses `xcopy` to copy **all** files from the extracted folder into `deps/`. Delete `deps\llama-server.exe` and re-run `install.bat`.

### 7. `llama-server.exe` window opens then closes instantly
**Cause:** The server is launched from the wrong directory and can't find its DLLs.
**Fix:** `start.bat` now uses `/D "deps"` to launch the server from its own directory. Ensure your `start.bat` is the latest version.

### 8. `Failed to establish a new connection` (chat error)
**Cause:** The llama.cpp server is not running.
**Fix:**
- Make sure you ran `start.bat`, not just opened the HTML file.
- Check that both the llama.cpp window AND the Python backend window are open.
- If the llama.cpp window shows an error, read it and refer to the fixes above.
- Try the **Manual Server Launch** steps above.

### 9. `pip install` fails (externally-managed-environment)
**Cause:** Ubuntu/WSL blocks pip install outside a virtual environment.
**Fix:** Use a virtual environment or add `--break-system-packages`:
```cmd
python -m pip install flask flask-cors requests huggingface_hub --break-system-packages
```

### 10. Port 8080 or 5000 already in use
**Cause:** Another program is using one of the ports.
**Fix:** `start.bat` automatically kills processes on these ports. If it still fails, restart your computer or manually find and close the program.

### 11. Model takes hours to download / fails
**Cause:** Slow internet or HuggingFace rate limiting.
**Fix:**
- Wait — the model is ~5-6 GB.
- If it keeps failing, delete the partial file in `models/` and try again.
- You can also download the model manually from HuggingFace and place it in `models/`.

## 🎯 Usage

1. Run `install.bat` once.
2. Run `start.bat` every time you want to use WormGPT.
3. Open `http://localhost:5000` in your browser.
4. Chat with WormGPT — no filters, no limits.

### Web UI Features
- **Chat** — real-time streaming responses
- **Settings** — edit system prompt and user profile (name, age, interests, context)
- **History** — export and reload past conversations
- **Config** — adjust temperature and max tokens on the fly
